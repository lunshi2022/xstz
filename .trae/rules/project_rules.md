# 项目规则

## 默认行为

对于所有任务，请遵循以下流程：

1. **首先调用相关技能**
   - 需求分析 → 使用 `brainstorming`
   - 实现任务 → 使用 `writing-plans` → `executing-plans`
   - Bug 修复 → 使用 `systematic-debugging`
   - 新功能 → 使用 `test-driven-development`
   - 复杂实现 → 使用 `subagent-driven-development`

2. **遵循技能流程**
   - 不要跳过技能中的任何步骤
   - 完成技能流程后再进行回应

3. **始终使用技能工具**
   - 在做出任何回应前，先调用 Skill 工具
   - 即使只有 1% 的可能性，也要调用技能检查
